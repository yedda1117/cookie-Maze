# 🍯 地形代价系统说明

## 功能概述

新增了可爱的饼干主题地形代价系统，让迷宫更有趣！

### 🎮 游戏设定

- **主角**: 姜饼人 🍪
- **目标**: 从起点到达终点
- **挑战**: 蜂蜜地会消耗体力值

### 🗺️ 地形类型

1. **🍪 普通地面** (奶油色)
   - 代价: 0
   - 可以自由通行

2. **🍯 蜂蜜地** (金黄色)
   - 代价: 1
   - 粘稠的蜂蜜会让姜饼人行动困难
   - 每经过一格蜂蜜地，体力值减少10点

3. **🍫 巧克力障碍** (棕色)
   - 完全无法通行

### 🎯 算法对比

#### Dijkstra / A* 算法
- ✅ **考虑地形代价**
- ✅ 找到**总代价最低**的路径
- ✅ 会尽量避开蜂蜜地
- 适合：需要节省体力的情况

#### BFS 算法
- ❌ **不考虑地形代价**
- ✅ 只保证**步数最少**
- ❌ 可能经过很多蜂蜜地
- 适合：只关心最短距离的情况

### 📊 统计信息

右侧面板新增显示：

1. **姜饼人体力** - 实时显示剩余体力
   - 绿色/黄色：体力充足
   - 红色闪烁：体力不足（<30%）

2. **总代价 🍯** - 路径的总代价
   - 包含基础移动代价 + 地形额外代价
   - Dijkstra/A* 会最小化这个值

### 🎨 视觉效果

- **蜂蜜地**: 金黄色渐变，带有光泽和水珠效果
- **体力条**: 动态变化，低体力时会闪烁警告
- **姜饼人图标**: 可爱的动画效果

### 🕹️ 使用方法

1. 点击"设置蜂蜜地"按钮（金黄色饼干按钮）
2. 在地图上点击设置蜂蜜地形
3. 设置起点和终点
4. 选择算法执行搜索
5. 观察不同算法的路径选择差异

### 💡 实验建议

尝试以下场景来体验算法差异：

1. **场景1**: 在起点和终点之间放置一条蜂蜜地带
   - BFS: 会直接穿过
   - Dijkstra/A*: 会绕路避开

2. **场景2**: 创建两条路径
   - 路径A: 短但全是蜂蜜地
   - 路径B: 长但是普通地面
   - 观察不同算法的选择

3. **场景3**: 随机分布蜂蜜地
   - 对比各算法的总代价差异

## 技术实现

### 后端改动
- `Node` 类新增 `terrain_cost` 属性
- `Grid` 类新增 `set_honey()` 方法
- Dijkstra/A* 算法计算代价时加入地形代价
- API 返回 `total_cost` 统计信息

### 前端改动
- 新增蜂蜜地形绘制（金黄色渐变 + 光泽效果）
- 新增体力值显示组件
- 路径动画时实时更新体力值
- 新增"设置蜂蜜地"按钮

### 配色方案
- 蜂蜜地: `#FFD700` → `#FFB84D` → `#FFA500`
- 体力条: `#FFD700` → `#FFA500` → `#FF8C00`
- 低体力警告: `#FF6B6B` → `#FF4444` → `#CC0000`
