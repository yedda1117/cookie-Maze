# 快速开始指南

## 📦 1分钟快速启动

### Windows用户

```powershell
# 1. 打开PowerShell，进入项目目录
cd path\to\MazeVisualizer

# 2. 创建虚拟环境
python -m venv venv

# 3. 激活虚拟环境
.\venv\Scripts\Activate.ps1

# 4. 安装依赖
pip install -r requirements.txt

# 5. 启动应用
python main.py

# 6. 打开浏览器访问
# http://127.0.0.1:5000
```

### macOS/Linux用户

```bash
# 1. 进入项目目录
cd path/to/MazeVisualizer

# 2. 创建虚拟环境
python3 -m venv venv

# 3. 激活虚拟环境
source venv/bin/activate

# 4. 安装依赖
pip install -r requirements.txt

# 5. 启动应用
python main.py

# 6. 打开浏览器访问
# http://127.0.0.1:5000
```

## 🎮 基本操作

### 第一次使用

1. **设置起点**
   - 点击左侧面板的"设置起点"按钮
   - 在地图上点击（推荐左上角）
   - 看到绿色方块表示设置成功

2. **设置终点**
   - 点击"设置终点"按钮
   - 在地图上点击（推荐右下角）
   - 看到红色方块表示设置成功

3. **设置障碍物**（可选）
   - 点击"设置障碍物"按钮
   - 在地图上点击多个位置创建障碍
   - 或点击"随机迷宫"自动生成

4. **执行搜索**
   - 从"搜索算法"下拉菜单选择算法
   - 点击"执行搜索"按钮
   - 观看搜索过程和结果

## 🐛 常见问题

### 问：启动时提示"端口被占用"
**答：** 有其他程序占用了5000端口
```bash
# 使用其他端口启动
python run.py --port 8080
# 然后访问 http://127.0.0.1:8080
```

### 问：网页打开是空白的
**答：** 检查浏览器控制台（F12）是否有错误
- 确认Flask服务器运行正常
- 尝试刷新页面（Ctrl+R）
- 检查网络连接

### 问：点击地图没有反应
**答：** 可能没有选择操作模式
- 确保点击了"设置起点"/"设置终点"等按钮
- 按钮应该有视觉反馈
- 在操作模式选择后再点击地图

### 问：搜索找不到路径
**答：** 这是正常的
- 起点和终点之间没有可通行的路径
- 检查是否有障碍物阻挡
- 点击"清除障碍物"重试

### 问：Live Share链接无效
**答：** 链接有时间限制
- 重新运行 `Live Share: Share Server`
- 生成新链接即可

## 关键概念速览

### 网格地图
- **绿色**：起点 (Start)
- **红色**：终点 (End)  
- **黑色**：障碍物 (Obstacle)
- **浅蓝色**：被访问过的节点
- **浅粉色**：被扩展过的节点
- **金黄色**：最终路径

### 算法选择建议
| 算法 | 适用场景 | 特点 |
|------|---------|------|
| BFS | 无权图 | 保证最短路径，速度较快 |
| DFS | 路径探索 | 不保证最短，内存少 |
| Dijkstra | 加权图 | 精确最短路径，速度中等 |
| A* | 有终点目标 | 最快，需要好的启发式函数 |

### 启发式函数比较
| 函数 | 说明 | 速度 |
|------|------|------|
| 曼哈顿 | 城市方块距离 | 最快 |
| 欧氏 | 直线距离 | 中等 |
| 对角线 | Chebyshev距离 | 快 |

## 查看统计信息

右侧控制面板的表格显示每次搜索的统计数据：

| 指标 | 含义 | 单位 |
|------|------|------|
| 路径长度 | 从起点到终点的节点数 | 个 |
| 访问节点数 | 被遍历的总节点数 | 个 |

**性能评估**：访问节点数越少，算法越高效。

## 故障排除

### 问题：窗口启动失败
**症状**：`ImportError: No module named 'PyQt5'`
**解决**：运行 `pip install PyQt5==5.15.9`

### 问题：点击按钮没有反应
**症状**：搜索按钮点击后没有动作
**解决**：
1. 检查是否设置了起点和终点（右侧应显示绿色和红色节点）
2. 尝试点击"清空地图"重新设置
3. 检查是否在搜索中（搜索期间按钮会被禁用）

### 问题：搜索速度过慢
**症状**：大网格搜索响应缓慢
**解决**：
1. 使用A*算法而不是BFS（搜索节点更少）
2. 调节"速度"滑块使得搜索更快（但显示可能不够清晰）
3. 减小障碍物数量

### 问题：找不到路径（所有算法都返回否）
**症状**：起点被完全被障碍物包围
**解决**：
1. 点击"清空地图"清除所有障碍物
2. 重新设置起点和终点
3. 确保起点和终点之间有可通行的路径

## 高级技巧

### 对比不同启发式函数
```
1. 保持同一个地图（不点击"清空地图"或"生成迷宫"）
2. 选择"A* (曼哈顿)"，点击"开始搜索"，记录统计数据
3. 点击"重置"按钮（保留地图）
4. 选择"A* (欧氏)"，重复搜索
5. 比较访问节点数：通常曼哈顿最优
```

### 手动设计测试用例
```
1. 点击"清空地图"
2. 手动绘制特定的障碍物配置（如：狭长通道、迷宫等）
3. 设置起点和终点
4. 用不同算法搜索
5. 观察和记录结果差异
```

### 验证最短路径
```
对于BFS和Dijkstra：返回的路径是最短的
验证方法：数一下金黄色节点的数量 = 路径长度
```
