小组分工说明（最终版）
一、分工原则

为保证工作量均衡、难度匹配、工程结构合理，本项目采用 “事件流 + 渲染分离” 的协作方式：

A 负责底层算法逻辑与部分迷宫生成算法

B 负责可视化渲染、UI 控制、以及部分高级算法

两人各自负责三类算法（含 DP / 分支限界等）

二、成员 A 最终职责
（一）迷宫生成（核心逻辑）

负责 DFS 迷宫生成算法

负责 回溯法（Recursive Backtracking）迷宫生成

输出迷宫生成的 事件流（挖墙、标记路径、访问节点等），供 B 实现 UI 渲染

（二）三类算法实现（A 负责三类）
1. BFS 算法

队列管理、节点扩展顺序

输出颜色变更事件（访问、入队、出队）

2. DFS 算法

基于栈的深度优先遍历

输出回溯事件，实现可视化的回溯过程

3. DP（动态规划）算法

负责最优路径或资源最优收集模型

输出 DP 填表事件，用于表格动画渲染

（三）文档职责

编写迷宫生成模块设计文档（数据结构、流程说明）

编写 BFS / DFS / DP 算法详细设计文档（伪代码、复杂度分析）

整理算法测试数据

（四）测试与联调

完成迷宫生成模块单元测试

对 BFS / DFS / DP 的正确性与稳定性测试

与 B 联调事件流传输，测试高频刷新与动画稳定性

三、成员 B 最终职责
（一）UI 主控制器与可视化渲染（核心逻辑）

使用 Tkinter / PyQt 构建 UI 主界面

绘制迷宫网格区域（cell 渲染、边界渲染）

控制算法播放逻辑：start / pause / step / reset

接收 A 的事件流并转换成动画（颜色变化、路径高亮、DP 表格动画）

（二）三类算法实现（B 负责三类）
1. Dijkstra 算法

使用优先队列（heapq）

输出路径更新事件、Open/Close 列表事件

2. A* 算法

实现曼哈顿 / 欧式 / 对角线三种启发式

维护 f/g/h 表并输出变化用于可视化

3. 分支限界算法（Branch & Bound）

设计剪枝规则（上界估计与节点过滤）

输出剪枝事件、搜索树节点事件供渲染

（三）文档职责

编写 UI 模块设计文档（界面定义、接口说明）

编写 Dijkstra / A* / 分支限界详细设计文档

编写 统计分析部分（扩展节点数、路径代价、运行时间）

（四）测试与联调

测试 UI 控件响应、动画刷新稳定性

测试不同迷宫规模下 Dijkstra 与 A* 的性能差异

与 A 联调事件流格式，确保 UI 与算法同步稳定

四、最终成果要求

六大算法完整实现：
DFS、BFS、DP、Dijkstra、A*、分支限界

迷宫生成动画完整可视化

路径搜索过程可视化（Open/Close、访问、回溯等）

DP 填表动画、剪枝树动画

完整文档与 PPT（含分工说明）
