自选题选题策略说明

一、选题名称（按老师给定题目风格）

题目7（自选题）：多算法路径规划可视化系统

在一个由网格（Grid Map）构成的环境中，通过布设障碍物、设置起点与终点， 使用多种路径规划算法（如 BFS、DFS、Dijkstra、A* 等）自动搜索最短路径， 并通过动态可视化方式展示不同算法在搜索过程中的差异。

本系统不仅可以展示各算法的路径搜索结果，还可以动态呈现算法的搜索过程 （如节点访问顺序、开放列表/关闭列表变化、搜索区域扩张情况等）， 让用户直观理解不同算法的工作原理与性能差异。

二、与老师固定题目的对比与优势

课程综合实训文件中提供了六个固定题目（选课系统、推理系统、数独、人机棋类对弈、杀人游戏、购物单）。 这些题目均能覆盖课程的部分算法思想，但在算法覆盖面、可视化展示效果、系统规模和创新空间等方面存在一定局限。

1. 固定题目的主要算法及局限性

固定题目 主要算法 / 策略 局限性
题目1：选课方案设计 课程依赖建模 + 搜索 / 简单动态规划 算法数量较少，可视化多为表格或简单树结构，展示效果一般。
题目2：决策系统 前向推理、后向推理 规则结构单一，可视化主要是规则链和文字过程，视觉冲击力有限。
题目3：数独游戏 回溯 + 搜索 + 剪枝 核心算法基本都是回溯类，算法种类较单一，扩展性不强。
题目4：人机棋牌游戏（五子棋、象棋等） 博弈树、Minimax、简单评估函数 AI 难度高且与本门课重点（通用算法策略）耦合不高，工程量大，风险大。
题目5：另类杀人游戏 约瑟夫环数学模型、循环链表 / 模拟 算法深度较弱，可视化多为简单圆圈演示，难以体现综合算法能力。
题目6：购物单 0-1 背包动态规划 核心算法主要是 DP，展示以列表/表格为主，可视化效果有限，创新空间较小。

2. 自选题“多算法路径规划可视化系统”的综合优势

（1）算法覆盖面广，契合课程核心要求

课程要求掌握递归、分治、动态规划、贪心、回溯、图搜索等多种策略。本系统中可以自然引入：

  • DFS（深度优先搜索）：体现递归与回溯思想。
  • BFS（广度优先搜索）：无权图最短路径搜索。
  • Dijkstra 算法：加权图单源最短路径，体现贪心策略与优先队列的使用。
  • A* 算法：启发式搜索，将贪心 + 最短路径 + 启发式函数融合。
  • 可选扩展：双向 BFS、IDA* 等提高搜索效率的优化算法。

与固定题目多集中于单一算法相比，本自选题能在一个统一平台中展示多种经典算法， 更充分体现“掌握各种算法策略基本思想”和“比较算法效率、优化算法”的课程目标。

（2）可视化展示效果强，适合课堂演示

固定题目在可视化方面普遍以表格、文本或简单界面为主，视觉效果相对有限。

本系统可实现：

  • 网格地图的动态图形展示（起点、终点、障碍物分区显示）。
  • 搜索过程可视化：搜索波前扩张动画，访问过的节点由浅到深着色。
  • 开放列表与关闭列表的直观展示（不同颜色或区域表示）。
  • 路径回溯动画：最终最短路径按顺序高亮显示。
  • 多算法之间的路径、搜索范围、耗时对比，以图形或图表形式展现。

这些可视化效果不仅便于老师和同学直观理解算法工作过程，也极大提升演示时的观感， 相比固定题目更适合作为综合实训答辩展示内容。

（3）系统结构完整，更符合“综合实训”要求

路径规划可视化系统天然包含多个模块，便于进行软件工程化设计：

  • 地图与网格数据结构模块（负责存储节点、障碍物及邻接关系）。
  • 算法模块（BFS、DFS、Dijkstra、A* 等独立封装）。
  • 可视化模块（绘制网格、节点状态、路径及过程动画）。
  • 交互模块（用户设置起点/终点、算法选择、参数调节）。
  • 统计与分析模块（记录并展示路径长度、扩展节点数、运行时间等）。

这种多模块、多层次的设计，更利于开展需求分析、概要设计、详细设计与测试调试等完整实训过程。

（4）创新空间大，便于体现小组特点

在基础路径规划的前提下，本项目可以进一步扩展：

  • 引入多种启发式函数，对比不同启发式下 A* 的性能差异。
  • 加入随机地图/迷宫生成器，与搜索算法结合形成完整“生成 + 求解”流程。
  • 提供算法性能统计与图表对比界面，以数据形式佐证算法分析。
  • 增加运行速度调节、步骤回放等人机交互设计，提升用户体验。

这些创新点都可以在汇报中重点展示，明显区别于固定题目，体现“自选题”的优势和小组的设计能力。

三、参考的 GitHub 项目说明

为保证项目的工程质量和可视化效果，我们将参考（而非照搬）以下开源项目的设计思路：

参考项目：Pathfinding Visualizer（Python）

项目主页示例（Python + Tkinter/Pygame 路径规划可视化）： https://github.com/techwithtim/Pathfinding-Visualizer

该开源项目的主要特点包括：

  • 实现了 A* 算法的基本可视化。
  • 支持网格绘制、起点终点设置与障碍物设置。
  • 可以动态展示单一算法的搜索过程。

其局限性主要有：

  • 算法种类较少，通常只演示 A* 或少数几种算法。
  • 界面相对简单，缺乏统计图表与交互控制面板。
  • 没有开放/关闭列表的详细可视化，算法内部状态展示不充分。
  • 缺少多算法对比功能（例如 BFS vs Dijkstra vs A* 的对比）。
  • 代码结构模块化程度有限，不完全适合作为教学级的工程案例。

因此，我们将以此类项目的总体思路为参考，独立重新设计系统架构和界面， 并自行实现多种算法及相关可视化功能，确保项目既有工程质量又有原创性。

四、在 GitHub 思路基础上的实际升级与改进

为避免简单照搬开源代码、同时保证项目的精度与可行性，我们计划进行以下“可实际完成”的升级改造：

1. 算法数量与种类升级

在参考项目通常仅实现单一 A* 算法的基础上，我们将扩展并独立实现以下算法：

  • BFS（Breadth-First Search，广度优先搜索）。
  • DFS（Depth-First Search，深度优先搜索）。
  • Dijkstra 最短路径算法（体现贪心策略与优先队列使用）。
  • A* 启发式搜索算法（核心算法之一）。
  • 可根据时间和难度，选择性加入双向 BFS 或 IDA* 等进阶算法。

这些算法将以独立模块形式实现，便于在报告中进行复杂度分析与效果对比。

2. 启发式函数多样化

在 A* 算法中，我们将设计并实现多种启发式函数，包括但不限于：

  • 曼哈顿距离（Manhattan Distance）。
  • 欧氏距离（Euclidean Distance）。
  • 对角线距离（Diagonal Distance）。

用户可在界面中选择不同启发式函数，观察和比较搜索路径及搜索效率的变化，从而更好地理解启发式对 A* 算法性能的影响。

3. 开放列表 / 关闭列表可视化

我们将对算法内部的关键数据结构进行可视化展示，例如：

  • 将“开放列表（Open List）”中的节点用特定颜色标记。
  • 将“关闭列表（Closed List）”中的节点用另一种颜色显示。
  • 可选显示节点的 f = g + h 值等信息。

相比参考项目只展示路径，本升级有助于完整呈现算法的中间过程，便于教学与讲解。

4. 统计与对比分析模块

我们将设计统计模块，对不同算法的运行情况进行量化：

  • 记录每次搜索扩展的节点数量。
  • 记录最终路径长度（步数 / 权值和）。
  • 记录算法运行时间（可以采用简单计时）。

并通过表格或简单图表（如柱状图）展示不同算法在同一地图上的表现差异，使算法复杂度分析更加直观和有说服力。

5. 全新 UI 设计与交互体验优化

界面部分我们将采用全新设计，而不是沿用开源项目的布局和风格：

  • 使用 Python 图形库（如 PyQt5、Tkinter 或其他合适方案）重新设计窗口布局。
  • 左侧为网格地图可视化区域，右侧为控制面板（算法选择、参数设置、开始/暂停按钮等）。
  • 在界面中加入运行速度调节、清空地图、一键生成随机障碍等交互控件。

通过自主设计的 UI 和交互逻辑，可以明显区别于开源项目，体现小组的界面设计能力和用户体验设计能力， 也降低老师认为“直接照搬代码”的风险。

6. 模块化与层次化结构重构

我们将从零开始规划项目目录和模块划分，例如：

  • core/：存放网格与节点数据结构、各类算法实现。
  • ui/：存放界面相关代码，如主窗口、画布控件、控制面板等。
  • utils/：工具函数模块，如计时、随机地图生成等。

这种分层结构不仅有利于代码维护，也更符合课程综合实训“概要设计”“模块划分”的要求， 便于在报告中清晰展示系统架构图和模块调用关系。

7. 地图生成与扩展功能（视时间和精力逐步实现）

在时间允许的情况下，我们会加入：

  • 随机地图 / 障碍物生成器。
  • 简单迷宫生成算法（如基于 DFS 或 Prim 的迷宫生成）。
  • 地图保存与加载功能，便于测试同一地图上的多算法对比。

这些功能在不大幅增加实现复杂度的前提下，能显著增强系统演示效果与实验可重复性。

五、总结

综上所述，“多算法路径规划可视化系统”在算法多样性、可视化效果、系统完整性和创新空间等方面， 均明显优于课程给定的六个固定题目。同时，我们只是参考 GitHub 上已有项目的设计思路， 而在代码实现、架构设计和界面交互等方面进行全面重构与升级，确保项目既具有工程质量，又具备充分的原创性和可展示性。

该选题既能满足课程综合实训对数据结构与算法应用的要求，又能通过可视化和多算法对比为课堂演示提供良好效果， 是一个可行性高、风险可控、同时具有较高上限和评分潜力的自选项目。
